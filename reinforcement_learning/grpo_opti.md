# DAPO (Decoupled Clip and Dynamic sAmpling Policy Optimization)

## 动机：GRPO存在的问题：

- [熵塌陷（Entropy Collapse）](entropy_collapse.md)
- 梯度消失：当GRPO的一组采样出现答案全对或者答案全错时，会出现两种极端问题：
    1. 答案全对，说明模型掌握了当前的instance（以一定概率置信），那么该样本为简单样本，无梯度且浪费计算资源。
    2. 答案全错，GRPO产生的所有组内优势为0，所以这些采样所带来的gradient也都是0，相当于跳过了全0的困难样本。
- 截断样本奖励噪声：指的是，模型在生成rollout的数据的时候，因为达到了最大长度限制而被强行截断，未能生成完整的结束符，导致奖励模型给出了不合理偏低的评分，进而通过GRPO的组内相对优势计算，污染了整个组的梯度信号的现象，影响了训练的稳定性。

## 实现：对应以上问题的解决方案
### 更高截断（Clip-Higher），解决熵塌陷：对token-level的重要性采样进行不对称截断。

在标准的GRPO或PPO损失函数中，模型会计算当前新策略与旧策略之间的重要性采样比率$r_t$。为了保证训练稳定，传统算法会给这个比率设定一个完全对称的截断范围，例如[1 - $\epsilon$, 1 + $\epsilon$]，通常$\epsilon = 2$，即截断在[0.8, 1.2]之间。

当某个动作表现优异的时候（优势函数A > 0）时，模型想要提高它的概率：
- 对于本身已经是高概率的token：假设其生成概率从0.8提升到0.96，比率$r_t = 1.2$，正好没有触发截断。这些被模型偏好的词可以顺畅地继续被强化。
- 对于罕见的“探索性”token： 假设模型偶尔尝试了一个绝妙的新思路，其生成概率从0.05提升到0.07，比率$r_t$瞬间达到1.4。此时多余的梯度会被截断。

结果：这种对称截断实际上在惩罚探索。

DAPO的不对称截断机制：

DAPO将重要性采样比率的上下限解耦：
- 抑制端保持不变：截断下限依然维持在$1 - \epsilon_{low}$。当动作表现极差时（A < 0），依然严格限制其概率下降的速度，防止模型过度遗忘或者训练崩溃。
- 鼓励端大幅抬高：将截断上限放宽到$1 + \epsilon_{high}$

### 动态采样 （Dynamic Sampling）
保证每次采样出来的回答reward不全是0或者1，如果采样出来的全是0或者1，就继续采样，直到不满足为止。这样保证了每个采样Group都具有足够的优势信息，用于更新梯度

### Token-Level 策略梯度消失损失
为了防止序列长度对优势的稀释，将GRPO目标函数中每个序列内的优势平均替换成该批次内所有token的平均，确保每个token对梯度更新的贡献是均等的，无论它来自长序列还是短序列。

### 过长回答的奖励塑造（Overlong Reward Shaping）
- 过长样本过滤 （Overlong Filtering）：被截断的样本屏蔽其损失
- 软过长惩罚机制 （Soft Overlong Punishment）：当回答长度超过预设的最大值的时候，会施加一个惩罚，且惩罚力度随长度增大而变大。

在消融实验上看，DAPO每个改进方法都有提升。

# Dr.GRPO：修正GRPO的优势估计偏差
Dr.GRPO是与DAPO同期的工作，所以还是面向GRPO的改进而非DAPO

Motivation：原始GRPO的优势估计有偏（Bias），和DAPO的Token-level策略梯度损失动机类似但是更进一步，具体来说：

- 响应长度偏差（Response-level length bias）：来自响应长度$|o_i|$项
当优势A > 0 （表示回答正确时），较短的回答会产生更大的梯度更新，从而让策略偏好更短的正确回答。
当优势A < 0 （表示回答错误时），较长回答会因为$|o_i|$较大而受到的惩罚更小，使得策略在错误回答中更偏好较长的回答。

- 问题难度偏差 （Question-level difficulty bias）：来自GRPO优势函数的标准差项
标准差较低的问题（如特别简单或者特别困难，几乎总是0或者总是1）会得到更大的权重，从而在策略更新中影响更大。与常见的advantage normalization（按batch归一化）不同，这种按问题归一化会让不同问题在目标函数中具有差异化的权重，导致难度相关偏置。

实现：移除以上讨论的两个偏差项：即在计算策略梯度时不除以回答长度$|o_i|$，在计算优势时不除以奖励的标准差。修正后的目标函数等价于标准的PPO目标，而优势项进一步简化。

后续改进GPG改进了Dr.GRPO的优势缩放问题（Dr.GRPO去掉标准差过于草率），进一步提升了GRPO的表现。

# GSPO （Group Sequence Policy Optimization）：面向MoE架构的重要性采样对齐

Motivation：之前的GRPO及其改进方法存在重要性采样和奖励颗粒度不对齐的问题，具体来说：

- GRPO的重要性采样都是Per-Token粒度的，但是Reward却是Sequence粒度的，两者不对齐。奖励模型通常是在模型完整输出一段对话之后，给出一个总分。我们不知道这句话里到底哪个token的贡献最大，只能把这个总分均摊给整个序列；损失函数在计算时，需要逐个对比新旧策略在每一个token上的概率比值。
带来的问题（噪声）：理论上，重要性采样需要多次采样来逼近真实的期望分布。但在语言模型逐词生成的过程中，对于特定的上下文前缀，我们通常只采样了一次具体的token。用单词采样的token级比率，去硬套一个宏观的序列级奖励，在统计学上会产生极大的方差。

另外一方面，重要性采样的有效性依赖于多次采样，但在token级别的单次采样会使重要性权重产生噪声。
- 为什么强调这个不对齐问题存在于MoE结构中：因为在MoE结构中，新旧策略可能会激活不同的专家，进一步增大Per-Token采样带来的噪声。

实现：采用sequence-level的重要性采样，对齐reward粒度。
